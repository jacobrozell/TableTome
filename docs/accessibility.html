Our commitment

Tabletome targets WCAG 2.1 AA engineering practices on core v1.0 flows: Play (guides and Guided Match), Rules reference, Models (Collection), and Settings.

Built-in iOS accessibility

  • VoiceOver — labels, hints, and identifiers on primary controls across Play, Rules, Models, and Settings.
  • Dynamic Type — semantic text styles scale with your system font size; key screens adapt at the largest accessibility sizes.
  • Touch targets — minimum 44×44 pt on primary controls.
  • Reduce Motion — guide and onboarding animations respect the system setting.
  • Dark Mode — system, light, and dark appearance in Settings.

Known limitations

  • English only.
  • Some dense battle-tracker layouts may require scrolling at the largest text sizes on smaller phones.
  • External PDF and web links open in Safari or system viewers outside the app.
  • Tabletome does not include video content; Captions and Audio Descriptions do not apply.

Feedback

If you encounter an accessibility barrier on a core task, please file an issue with the screen name, assistive feature, and iOS version: GitHub Issues
